Software Engineering Concepts

Software Engineering Practices

  • Planning changes: Creating plans and ideations to create pitches for the class.
  • Checklists: To-do lists that ensure essential tasks are completed.
  • Burndowns: Visual progress tracking of tasks versus time.
  • Coding with comments: Writing explanatory notes to clarify the code, and comunicate with others
  • Help documentation: Showing understanding and using the codebase.

Software Development Lifecycle Practices

  • Source control: Tracking and managing changes to code using Git or similar tools.
  • Forking: Creating a copy of the project to work independently on it.
  • Branching: Developing code to later merge back into the main branch
  • Building: Compiling and packaging code into a usable program.
  • Testing & verification: Ensuring the program works as expected with verification make tools.
  • Pull requests: Submitting changes for review before merging into the main project.
  • Merging/integrating: Combining branches into the main codebase.
  • Deployment: Publishing software to a production environment.

Retrospective Engineering Practices

  • Presentation: Showing your work to others in a formal format.
  • Live reviews: Real-time feedback sessions with peers or mentors.
  • Demos: Interactive showcase of working examples of your project or features.
  • Code reviews: Checking each other's code for quality and correctness.
  • Revising plans: Updating your goals and strategy based on feedback.

Data Types

  • Numbers: Integers and decimals for calculations.
  • Strings: Text values in quotes.
  • Booleans: True/false values.
  • Arrays: Lists of ordered elements.
  • JSON objects: Key-value pairs for structured data.

Operators

  • String operations: Combining or modifying text values.
  • Mathematical operations: Arithmetic like addition, subtraction, multiplication, division. As well as; greater than; less than; equal to expressions; and trig.
  • Boolean expressions: Logical comparisons like &&, ||, ==.

Control Structures

  • Iteration: Loops like `for`, `while` that repeat tasks.
  • Conditions: `if/else` logic to run code selectively.
  • Nested conditions: Conditions within conditions for complex logic.

Input / Output

  • HTML5 input: Text boxes, checkboxes, and other user inputs.
  • Validation: Ensuring input data is correct.
  • Key events: Detecting keyboard input in real-time.
  • DOM: Accessing and modifying webpage elements with JavaScript.

Classes

  • Writing classes: Creating reusable object blueprints.
  • Creating methods: Functions that belong to classes.
  • Instantiating objects: Making usable objects from classes.
  • Using objects: Accessing and modifying data inside objects.
  • Calling methods: Running the behavior of the class.
  • Parameters: Input values for methods or functions.
  • Return values: Outputs from functions or methods.

Coding Practices

  • SRP (Single Responsibility Principle): Each function/class should do one thing well.
  • Object Literal: Defining objects directly using curly braces.
  • Object Instance: Objects created from class blueprints.
  • FSMs in Game (Finite State Machines): Managing game states like menus, battles, levels.
  • Inheritance: Extending classes to reuse code and structure.
